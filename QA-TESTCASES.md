# QA 테스트케이스 & 결과 (v9.33)

검증 방법: 스크래치패드 로컬 사본(키 blank=LocalSync, 백엔드 쓰기 없음)에서 Chrome 자동화로 플로우 실행 + 라이브는 읽기 전용 curl. 실데모/실셀에는 절대 쓰기 안 함.

## 이번 회차에 발견·수정한 버그
| # | 증상 | 원인 | 조치 |
|---|---|---|---|
| B1 | 전체 메뉴판 스크롤 시 헤더 위로 메뉴가 살짝 비침 | `.sheet padding-top:8px` 때문에 sticky 헤더가 8px 아래에 붙어 그 틈으로 목록이 보임 | `.sheet` 상단 패딩 제거(그립 마진이 여백 유지) + 헤더 좌우 bleed·z-index↑ → 빈틈 0 |
| B2 | 스크롤된 시트를 닫고 다른 시트를 열면 새 시트가 스크롤된 채로 뜸 | `openSheet`가 `scrollTop`을 리셋 안 함 | `openSheet`에 `scrollTop=0` 추가 |
| B3(경미) | `member_diff` 함수가 anon에서도 실행됨 | Supabase 기본권한이 함수 생성 시 anon에 EXECUTE 부여 → `revoke from public`만으론 부족 | `revoke ... from public, anon, authenticated` (순수 함수라 무해하나 최소권한 정리) |
